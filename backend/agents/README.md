# TripCraft AI - 代理架构

TripCraft AI 使用由Agno驱动的复杂多代理系统来创建个性化旅行体验。本文档解释了系统中的不同代理及其角色。

## 团队结构

系统由"TripCraft AI团队"协调，该团队协调多个专业代理来创建全面的旅行计划。团队以协调模式运行，确保旅行规划的所有方面都得到高效处理。

### 核心团队成员

1. **目的地探索者**
   - 主要角色：研究和推荐旅游景点和体验
   - 工具：用于深度网络研究的ExaTools
   - 重点领域：
     - 著名地标和纪念碑
     - 热门旅游景点
     - 博物馆和文化遗址
     - 购物区域
     - 家庭友好活动
   - 提供关于景点的结构化信息，包括开放时间、费用和游览时长

2. **酒店搜索代理**
   - 主要角色：住宿研究和推荐
   - 专注于基于以下条件寻找完美住宿：
     - 位置偏好
     - 预算限制
     - 所需设施
     - 房间类型
     - 物业特色

3. **餐饮代理**
   - 主要角色：餐厅和美食体验推荐
   - 考虑因素：
     - 菜系类型
     - 价格范围
     - 饮食限制
     - 氛围和环境
     - 位置和可达性
     - 特殊餐饮体验

4. **预算代理**
   - 主要角色：财务规划和成本优化
   - 职责：
     - 旅行成本分解
     - 预算分配
     - 节省成本建议
     - 货币考虑
     - 应急基金规划

5. **航班搜索代理**
   - 主要角色：航空旅行规划和优化
   - 处理：
     - 航班路线研究
     - 航空公司比较
     - 时刻表优化
     - 转机规划
     - 机场接送协调

6. **行程专家**
   - 主要角色：创建详细的逐日旅行时间表
   - 专业知识：
     - 逐小时活动规划
     - 景点优化时间安排
     - 交通调度
     - 现实旅行时间
     - 缓冲时间管理
     - 天气适应性调度
     - 旅行者特定节奏

## 团队协调

团队通过复杂的协调系统协同工作：
1. 分析用户偏好和需求
2. 将任务委派给专业代理
3. 将各个代理的输出组合成连贯的旅行计划
4. 确保旅行的所有方面都得到适当同步
5. 在所有决策中保持预算一致性

## 工具和技术

代理使用各种工具，包括：
- **ReasoningTools**：用于逻辑决策和计划优化
- **ExaTools**：用于深度网络研究和信息收集
- **FirecrawlTools**：用于实时数据和当前信息

## 输出格式

团队产生详细的旅行行程，包括：
- 旅行执行摘要
- 全面的旅行物流
- 逐日行程
- 详细的住宿信息
- 精选体验和活动
- 完整的预算分解

## 最佳实践

代理系统遵循这些关键原则：
1. 彻底分析用户偏好
2. 使用多个数据源进行详细研究
3. 实用且可实施的推荐
4. 备用选项和应急计划
5. 清晰的沟通和结构化输出
6. 在所有决策中保持预算意识

## 集成

此代理架构设计为与TripCraft AI后端无缝协作，为创建既神奇又实用的个性化旅行体验提供强大的基础。
